The Math Behind Bunking

The "Safe Bunk" Formula

Bunkit calculates how many future classes you can skip based on the total projected classes in the semester.

STILL NEED ceil( (Criteria% * FinalTotal) - Attended )
MAX SKIPPABLE max(0, Remaining - StillNeed)
graph TD A[Input: Attended, Held, Remaining] --> B{Calc FinalTotal} B -- Attended + Held + Remaining --> C[Apply Target Criteria 75%] C --> D[Still Need Calculation] D --> E[Max Safe Bunks Calculated]

Optimal Break Projection

A proprietary greedy algorithm that scans the entire semester timeline to find the longest possible stretch of holidays by strategically "spending" your safe bunk budget on working days.

Simulation Sequence:

  1. 1 Build a timeline of all days until semester end.
  2. 2 Identify weekends and pre-defined holidays as "Free Days".
  3. 3 Run a sliding window to find max consecutive days by filling gaps with "Safe Bunks".
graph LR T[Timeline] --> WD[Working Days] T --> HD[Holidays] WD --> BW[Sliding Window] BW --> LB[Longest Possible Break Found]

💰 The Monetization Engine

The BunkitAdManager (js/ad-manager.js) is a custom-built, non-intrusive monetization pipeline that loads dynamic configurations (like image URLs and CTA links) directly from the app_config database. It features:

  • ⏱️
    Dynamic Skip Timers

    Ads implement a forced skip-delay (e.g., 4s) to ensure visibility before allowing dismissal, ensuring high impressions.

  • 👑
    Premium User Bypass

    The system actively cross-references the AuthManager.user.email against a premium_users config node. If a valid, unexpired premium pass is found, the generation sequence is synchronously aborted—providing a 100% ad-free experience.

⚡ Offline Architecture & Smart Reminders

BunkIt operates as a fully functional Progressive Web App (PWA) driven by a robust Service Worker (sw.js).

Caching Strategies

Implements Stale-While-Revalidate for external assets (like Chart.js) and a Network-First (3s Timeout) strategy for core HTML pages, falling back to cache or an offline page instantly if the network fails.

Context-Aware Reminders

The Service Worker autonomously analyzes local IndexedDB data hourly. It calculates low-attendance thresholds and logging streaks, utilizing native showNotification and scheduled TimestampTrigger APIs to nudge users intelligently without a server.
