The Problem Statement

  • Strict Penalty Systems: Universities enforce mandatory attendance (75%), dropping grades heavily for shortages.
  • Static University Portals: Existing ERP systems (like Academia) only provide current static data, but zero predictive analytics regarding future margins.
  • Erroneous Manual Tracking: Students resort to mental math or excel sheets, leading to critical miscalculations resulting in debarment.
  • Communication Gaps: Lack of centralized platforms to instantly notify peers about sudden class cancellations or mass bunk coordinations.

The Solution: BunkIt

An Intelligent Progressive Web App (PWA) with Offline-First capabilities and Mathematical Forecasting.

Predictive Math

Offline Active

Core Algorithm: Safe Bunks

Calculating absolute boundaries using integer sequences.

  • The mathematical engine calculates exact future classes you can skip based on the Total Projected Classes for the semester.
  • Classes Still Needed = Math.ceil( (Target% * FinalExpectedTotal) - AlreadyAttended )
  • Max Safe Bunks Buffer = Math.max(0, TotalRemaining - ClassesStillNeeded) 

// Predictive Forecasting
let currentPresent = classData.present;
let currentTotal = classData.present + classData.absent;

let safeBunks = 0;
// Iteratively project future bunks until boundary is crossed
while ( ((currentPresent) / (currentTotal + safeBunks + 1)) * 100 >= targetPercentage ) {
    safeBunks++;
}

return safeBunks;

Algorithm: Long Weekend Projection

A proprietary greedy algorithm mapping calendar constraints.

1. Timeline Mapping

Constructs a comprehensive array of all dates from `today` until the calculated semester end date.

2. Identify Free Days

Flags all natural Weekends (Saturdays/Sundays) and pre-declared University Holidays inside the array.

3. Strategic Bridging

Uses a Sliding Window Sequence to "spend" your mathematical Safe Bunks on working days bridging two holidays.

[Thu: Holiday] → [Fri: BUNK] → [Sat: Wknd] → [Sun: Wknd]

Result: 4-Day Continuous Break for 1 Safe Bunk.

Deep Dive into VAPID Push Architecture

Zero-SDK VAPID Push

BunkIt deliberately avoids bulky Firebase messaging SDKs (which double notification bugs and bloat clients). Instead, it implements raw VAPID (Voluntary Application Server Identification).

  • Guest Identity System: Dynamically generates Pseudo-Emails (e.g., `guest_8f29ax@bunkit.local`) allowing unauthenticated visitors to successfully register their `device_id` into the Postgres Database.
  • Edge Crypto: Supabase Edge Functions (`send-push.ts`) manually encrypt JSON payloads using Elliptic Curve Cryptography (`aes128gcm`) before transmitting directly to browser vendor endpoints (FCM/Mozilla/Apple).
  • Automated PostgreSQL Pipelines: Extension `pg_cron` schedules a daily background query that invokes `pg_net` POST requests to the Edge server exactly at 5:00 PM for daily Portal sync reminders.
