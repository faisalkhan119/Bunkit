PROJECT SYNOPSIS

1. Title of the Project

BunkIt – A Smart Attendance Analytics & Student Management Platform

2. Working/Methodology of the Project

BunkIt is an intelligent, offline-first web application designed to alleviate the stress of university attendance tracking. The methodology is built upon predictive database analytics and real-time syncing:

  • Data Collection & Flow: Students input their current attendance statistics or initialize a fresh semester. The application caches this data locally using IndexedDB for instant, offline access, and simultaneously syncs it to a remote PostgreSQL database to prevent data loss.
  • Predictive Analytics Engine: As daily attendance is logged (Present, Absent, Cancelled, or Holiday), the core algorithm mathematically calculates the user's current percentage. It dynamically computes two critical metrics: "Safe Bunks" (the exact number of consecutive classes a student can miss before falling below the mandatory threshold) and "Classes Required" (the precise number of consecutive classes a student must attend to recover a deficit percentage).
  • Advanced System Methodology: The platform includes a "Mass Bunk Polling" system where students sharing the same class ID can vote in real-time. This leverages PostgreSQL database webhooks to securely broadcast native Push Notifications to the specific student cohort.

3. Selected Frontend and Backend Technologies

The project utilizes a modern decoupled architecture:

  • Frontend Technologies (Client-Side): HTML5, Modern Vanilla JavaScript (ES6+), and Advanced CSS3. It utilizes Native Web Service Workers, Cache API, and IndexedDB for Progressive Web App (PWA) offline capabilities. The administrative dashboard is built using React.js and Tailwind CSS.
  • Backend Technologies (Server-Side & Database): Supabase (PostgreSQL) is utilized as the primary database with strict Row Level Security (RLS) policies. Serverless compute is handled by Edge Functions (Deno/TypeScript), and background task execution is managed via pg_cron. Real-time communication is facilitated by the Native Web Push API (VAPID protocol).

4. Expected Outcome of the Project

The primary outcome of BunkIt is to deliver a highly accessible cross-platform application that empowers students to proactively manage their academic standing. By providing dynamic mathematical forecasting rather than just static logs, the project will significantly reduce the number of students facing attendance-shortage penalties. Furthermore, it will demonstrate a scalable, offline-capable database architecture that provides a seamless user experience regardless of network connectivity.
